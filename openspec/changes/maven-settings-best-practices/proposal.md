## Why

The Maven `settings.xml` generated by the workstation setup tool is minimal — it only includes a mirror block and optional server credentials. This omits several sections that are standard in enterprise and team environments and leaves users with an incomplete configuration that doesn't reflect what they'd find in a well-maintained developer workstation setup.

## What Changes

- Expand the generated `settings.xml` to include all sections commonly found in industry-standard Maven configurations:
  - `<mirrors>` — internal mirror with correct `mirrorOf` value (e.g., `*,!local` instead of `*`)
  - `<servers>` — credentials using Maven's `${env.VAR}` interpolation pattern instead of plaintext placeholders
  - `<profiles>` — a named profile with `<repositories>` and `<pluginRepositories>` blocks pointing to the internal repo, with explicit release/snapshot policies
  - `<activeProfiles>` — activates the above profile by default
  - Proper XML namespace and `xsi:schemaLocation` declaration
- Update the credential placeholder strategy to use environment variable interpolation syntax (`${env.MAVEN_USERNAME}`, `${env.MAVEN_TOKEN}`) rather than literal placeholder strings (`YOUR_USERNAME`, `YOUR_TOKEN`)

## Capabilities

### New Capabilities

- `maven-settings-structure`: Defines the full structure and section requirements of a best-practice `settings.xml`, including mirrors, servers, profiles, activeProfiles, and credential handling conventions.

### Modified Capabilities

- `repository-config-generator`: The scenario "Maven configuration generated" must be updated — the output now includes profiles, activeProfiles, and namespace-correct XML in addition to the mirror block.

## Impact

- `workstation-setup/src/modules/maven.ts` — `configTemplate` function updated to emit the expanded settings structure
- Existing snapshot and script export behavior is unaffected; only the generated XML content changes
- No breaking changes to the UI or other package manager modules
